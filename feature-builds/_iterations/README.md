# Iterations

Lightweight notes for small changes that don't earn a full feature folder — no schema change, no new surface, no blast radius worth a full lock. Things like copy tweaks with a decision rationale, small prompt adjustments, minor config changes where you want a written record.

Convention: `YYYY-MM-DD-slug/ITERATION-NOTE.md`

If the change needs a pre-build lock, it goes in a named feature folder instead. When in doubt: if you'd want to reference the decision six months from now, write an iteration note.
