# UX reference

Consolidated mocks, production screenshots, and visual reference for the current product state. Use this folder as the source of truth for what the product looks like right now — not what it's supposed to look like, what it actually looks like.

Useful for:
- Side-by-side diffs in pre-build locks (V(N-1) reference)
- Onboarding new contributors to the current UI state
- Verifying that untouched surfaces stayed unchanged after a deploy

Convention: `YYYY-MM-DD-[feature-or-screen]-[state].png` for screenshots, `[screen]-mock.html` for HTML mocks.
